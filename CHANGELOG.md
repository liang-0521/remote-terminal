# 更新记录

## 0.5.1 - 2026-07-17

终端操作、多工作区并发和文件传输可靠性更新。

- 终端右键菜单新增复制、粘贴和全选，支持菜单键、Shift+F10、方向键及 Esc/Tab 关闭；复制保留多行、中文和空白内容。
- 多服务器并行操作按会话隔离排队写入和异步粘贴，切换或重连后不会把旧操作发往新会话；后台连接完成也不会从弹窗或输入框抢走焦点。
- 断线后保留远程目录快照与原路径，重连时按原路径刷新；迟到的旧会话事件和目录响应不会覆盖当前工作区。
- 下载任务在传输列表中持续展示排队、进度、速度、完成与失败状态，并纳入退出和更新安装前的活动任务检查。
- Windows 发布工作流统一复用 Rust 构建目录，并启用 npm 离线优先缓存参数，减少重复编译和依赖检查耗时。
- 安装包仍未使用受信任的 Authenticode 证书签名，Windows 可能显示“未知发布者”；Tauri updater 签名与 SHA-256 完整性校验继续强制执行。

## 0.5.0 - 2026-07-16

远程文件工作区、监控布局和交互可靠性更新。

- 资源管理器支持左侧、右侧和下方停靠；下方远程文件与传输任务作为同级标签共享高度，切换到服务器连接或性能监控时不再消失。
- 性能监控移到左侧工作区，刷新间隔可选 1、2、5、10、30 秒；未打开服务器时，资源管理器和性能监控会显示明确入口。
- `.log`、`.sh` 等远程文本文件支持双击预览与编辑、多文件标签、大小写/全字/正则搜索；活动日志采用增量读取，终端缓冲区在切换后继续保留。
- 未保存编辑在关闭文件、关闭服务器工作区和退出客户端前统一使用应用内确认框拦截；取消后保留标签、内容和脏标记。
- 同名上传必须明确确认覆盖，原生端先备份远程文件再替换；取消不会启动上传，失败任务保留在传输列表并显示原因。
- 远程文件右键菜单补充新建文件和文件夹，并修复矮窗口中菜单被面板裁切的问题；菜单会按窗口边界自动向上或向左避让。
- 修复已完成传输事件反复触发目录刷新、底部资源标签随左侧导航消失，以及右键菜单和文件列表持续闪动的问题。
- 终端断开后保留已有输出，按 Enter 可重连；行内命令提示最多显示五条，并随输入逐步缩小候选。

## 0.4.2 - 2026-07-15

远程文件下载流程更新。

- 远程文件右键菜单改为“下载到…”，通过 Windows 保存对话框直接选择本机路径。
- 下载先进入受管缓存并校验文件大小，再原子写入用户选定的目标，不留下半个文件。
- 移除原有的“下载并拖到电脑”交互及对应原生拖拽依赖，降低 Windows 文件操作的不确定性。

## 0.4.1 - 2026-07-15

界面可读性、状态恢复与命令提示更新。

- 修复浅色主题下远程文件右键菜单和关闭主窗口弹窗背景、边框与文字颜色不匹配的问题。
- 退出时保存窗口位置、大小和最大化状态，并持久化界面主题、强调色、终端颜色、资源管理器宽度和底部面板布局。
- 设置新增命令提示显示方式，可选择输入时自动显示，或仅通过命令按钮与快捷键打开。
- 输入完整命令后继续保留候选，并同时显示可继续拼接的更长命令，例如 `clear` 与 `clear_console`。

## 0.4.0 - 2026-07-15

工作区交互、命令辅助与数据存储设置更新。

- 活动栏固定保留，并支持“仅图标”和“图标加文字”两种宽度；服务器选择器补充明确的展开状态，空列表时直接进入新增服务器。
- 远程文件区支持横向调整宽度，监控、传输和问题页共享同一底部面板高度，切换标签不再跳变。
- 修复终端底部黑边、重复行内候选、空搜索混入大量远端命令，以及桌面窗口出现浏览器刷新菜单和刷新快捷键的问题；完整输入精确命令后不再继续显示行内候选。
- 命令历史改为按服务器保存在本机并支持删除；远端可执行命令与本机历史明确区分，远端结果不可删除，中文用途和英文命令搜索继续保持本地处理。
- 远程资源管理器改用右键菜单提供真实的打开、下载/拖到电脑、重命名/移动、删除和刷新操作；删除不递归，目录只有为空时才能删除。
- 支持把远程普通文件安全缓存后拖到 Windows 资源管理器，上传仍直接拖入远程文件列表，并保留传输进度、取消和重试。
- 移除连接分组；设置改为外观、数据与存储、应用多页结构，补齐强调色与终端颜色预设。
- 默认数据目录调整为安装可执行文件同级的 `data` 文件夹，并允许在设置中迁移到其他本机目录；迁移仅复制客户端管理的数据文件，保留原目录且不复制凭据或 WebView2 缓存。
- 真正退出且仍有活动 SSH 会话或传输时只显示一次合并提醒；隐藏到托盘不提醒，也不会中断现有工作区。
- 前端组件按工作区框架、连接、文件、终端、监控、设置和共享组件分层整理；回归测试与可复用 fixture 保留，QA 临时配置、截图、日志和生成物验证后不入库并精确清理。
- 当前没有可信 Authenticode 代码签名证书，Windows 安装包继续显示未知发布者且发布流程不再检查 Authenticode；Tauri updater 签名与 SHA-256 校验仍保持强制。

## 0.3.1 - 2026-07-15

Windows 安装包信任链准备与更新校验强化。

- 为后续版本接入 SHA-256 Authenticode 签名、可信时间戳和发布者证书身份复验能力。
- `0.3.1` 经项目所有者明确授权为一次性未签名例外；安装时 Windows SmartScreen 仍可能显示未知发布者提示。
- Tauri updater 签名、公开更新元数据和 SHA-256 校验继续强制验证，未降低客户端更新包的完整性校验。

## 0.3.0 - 2026-07-14

命令辅助与界面可读性更新。

- 将命令辅助拆分为两个独立入口：按钮打开支持中文用途和英文命令/意图的命令搜索，终端输入则在光标附近显示紧凑候选。
- 行内候选优先匹配实际命令前缀，修复输入 `ll` 时混入无关语义结果的问题；保留 Tab 插入、Enter 执行和 Escape 关闭的明确边界。
- 增加系统、亮色和暗色三种会话级界面主题，同时保持终端背景、文字颜色与背景图设置独立。
- 修复终端内容被底部面板遮挡、文件名过长显示不全及候选面板短结果留白过多的问题。

## 0.2.0 - 2026-07-14

首个低内存 Windows 原生架构版本。

- 将桌面外壳从 Electron 迁移到 Windows-only Tauri 2，保留现有 IDE 风格工作区；同机三轮可见空闲私有内存中位数降低约 31%。
- 补齐真实密码 SSH、主机指纹确认与变更硬阻断、交互式 PTY 终端，以及真实 SFTP 浏览、拖拽上传、取消和重试。
- 将“命令模板”升级为本地智能补全：合并远端命令、会话历史、中文语义和当前远程目录项，支持 `ll` 等常用别名，插入前不自动执行。
- 支持关闭窗口时每次询问、转入系统托盘后台运行或直接退出；活动传输期间退出与安装更新均会二次确认或阻止。
- 修复监控面板空白问题；顶部保留 CPU、内存、Swap 摘要，底部展示系统、负载、进程、网络和文件系统容量。
- Windows 安装器支持选择安装目录；从 `0.2.0` 起使用签名更新清单和 GitHub Releases 热更新。
- 保存密码仍为可选项，认证成功后才写入 Windows Credential Manager；旧版密码不迁移，需要首次重新输入。

升级说明：`0.1.0` 使用 Electron 更新器，无法安全验证跨框架静默替换；请手动安装一次 `0.2.0`。连接与主机指纹会自动迁移，此后版本可使用客户端内签名热更新。

## 0.1.0 - 2026-07-14

首个公开 Windows 版本。

- 提供密码 SSH、主机指纹确认、xterm 交互终端与本地命令模板。
- 支持用户主动选择后以 Windows `safeStorage` 加密保存密码，默认关闭、认证成功后才保存，并可单独清除。
- 补齐终端复制粘贴/中断键位、命令模板键位，以及服务器菜单、工作区、文件、传输和底部面板的键盘操作与列表语义。
- 提供 SFTP 目录浏览、文件列表拖拽上传、进度、取消和重试。
- 提供 CPU、内存、Swap、进程、网络和文件系统监控。
- 提供独立服务器工作区、可调整底部面板和会话级外观设置。
- 提供 Windows NSIS 安装器，以及基于 GitHub Releases 的后台检查、下载和重启更新能力。

已知边界：当前安装器未进行 Windows 代码签名；完整更新升级闭环将在发布更高版本后验证。
