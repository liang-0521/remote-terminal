# Remote Terminal

> **纯 AI 开发项目**：产品定义、界面设计、编码、测试、打包与发布均由 AI 完成。

Remote Terminal 是一款面向 Windows 的 Linux 远程服务器客户端，提供 SSH 终端、智能补全、SFTP 文件传输和性能监控。界面采用终端优先的 IDE 式布局，适合同时管理多台服务器和日常运维操作。

## 核心功能

- **SSH 终端**：支持密码认证、交互式终端、终端尺寸同步、主机指纹确认与指纹变化阻断。
- **智能补全**：结合远端可执行命令、本机按服务器保存的命令历史、中文语义和当前远程目录生成候选；支持 `ll`、`ls` 等常用命令，候选只插入、不自动执行，完整输入精确命令后自动关闭行内候选。
- **SFTP 文件管理**：浏览远程目录，直接拖放本机文件到列表上传；右键可执行打开、下载/拖到电脑、重命名/移动、删除和刷新，并提供传输进度、取消和重试。
- **服务器管理**：清晰的新增服务器入口，每台服务器使用独立工作区，切换后保留终端、目录、传输和监控状态。
- **性能监控**：顶部展示 CPU、内存和 Swap 摘要；底部展示系统负载、进程、网络趋势及文件系统容量。
- **后台运行**：关闭主窗口时可选择每次询问、隐藏到系统托盘或直接退出，后台传输和 SSH 会话可继续运行。
- **可选密码保存**：用户名默认 `root`；密码仅在用户选择且认证成功后保存到 Windows Credential Manager。
- **个性化设置**：支持强调色、终端背景色、文字颜色和终端背景图片；设置按外观、数据与存储、应用分页面展示。
- **可配置数据目录**：默认使用安装可执行文件同级的 `data` 文件夹，也可在设置中迁移到其他本机目录。
- **安装与更新**：Windows 安装器支持选择安装目录，并通过 GitHub Releases 提供由 Tauri updater 签名校验的热更新；当前安装包没有 Windows 发布者签名，安装时可能显示“未知发布者”。

## 界面布局

- 左侧活动栏：始终可见，左上角菜单仅切换“仅图标”和“图标加文字”；包含服务器、文件、传输、监控，以及固定在底部的设置入口。
- 远程文件区：展示当前服务器目录，同时作为 SFTP 拖放上传目标。
- 中央工作区：以终端为主，每台服务器对应独立标签页。
- 底部面板：传输、问题和监控共享同一个用户调整后的高度，可收起或展开，切换标签时不会跳变。
- 顶部栏：仅保留当前服务器和 CPU、内存、Swap 简要状态。

## 数据存储与迁移

- 默认数据目录为 `remote-terminal.exe` 所在目录下的 `data` 文件夹；设置中的“数据与存储”页面会同时显示当前路径和默认路径。
- 用户选择新的本机绝对路径后，客户端只迁移自己管理的持久化 JSON 数据；Windows Credential Manager 中的密码、WebView2 缓存和其他文件不会被复制。
- 从旧默认目录首次升级时采用同样的受控迁移，并保留旧目录作为可恢复来源；迁移完成后需重启才能使用新目录。已配置目录失效时会明确报错，不会静默切换到另一份数据。
- 命令历史按服务器保存在当前数据目录中，只在本机使用并可单条删除；远端可执行命令来自服务器能力探测，不属于本机历史，因此不能删除。

## 远程文件操作边界

- 文件列表本身是上传拖放目标，不常驻额外的上传卡片；只有拖入文件时才显示轻量提示。
- 右键菜单中的下载会先把远程文件安全缓存到本机，再允许拖到资源管理器；远程路径不会作为本机路径暴露。
- 重命名和移动都通过 SFTP 在远端执行。删除不递归：普通文件和符号链接可删除，目录仅在为空时可删除，非空目录会被明确拒绝。
- 真正退出应用且仍有活动 SSH 会话或文件传输时，只显示一次合并提醒；隐藏到系统托盘不会提醒，也不会中断工作区。

## 技术栈

- Tauri 2（Windows 原生桌面外壳）
- Rust（SSH、SFTP、监控、凭据与更新前检查）
- React + Vite
- xterm.js
- Recharts
- WebView2

## 项目结构

```text
remote-terminal/
├─ src/                       # React/WebView2 界面与原生能力调用
│  ├─ components/
│  │  ├─ shell/               # 顶栏、活动栏、工作区和底部面板框架
│  │  ├─ connections/         # 服务器管理与连接交互
│  │  ├─ files/               # 远程文件列表、拖放和右键操作
│  │  ├─ terminal/            # xterm 终端与命令辅助
│  │  ├─ monitoring/          # 监控视图
│  │  ├─ settings/            # 多页设置
│  │  └─ shared/              # 共享界面组件
│  └─ native/                 # Tauri 命令与事件适配
├─ src-tauri/src/             # Rust 原生核心
│  ├─ ssh.rs                  # SSH、SFTP、终端与传输
│  ├─ backend.rs              # 原生业务编排与并发边界
│  ├─ storage.rs              # 连接、主机指纹等持久化
│  ├─ command_history.rs      # 本机命令历史
│  ├─ data_directory.rs       # 数据目录校验与迁移
│  ├─ lifecycle.rs / tray.rs  # 退出、托盘与生命周期
│  └─ monitor.rs              # 远端性能监控解析
├─ test/                      # 可维护的前端/契约回归测试
├─ test-support/              # 可复用测试 fixture
├─ scripts/                   # 构建、发布和真实链路验证脚本
└─ .github/workflows/         # Windows 构建与 updater 签名发布流程
```

QA 临时配置、临时服务器、截图、日志和生成物不属于源码，验证结束后会按精确路径清理，也不会提交到仓库。

## 当前支持范围

- 客户端：Windows 10 / 11 x64
- 服务端：提供标准 SSH 与 SFTP 的 Linux 服务器
- 认证方式：密码认证
- 文件传输：普通文件上传、下载及拖到 Windows 资源管理器

暂不支持私钥、SSH Agent、跳板机、端口转发、目录递归上传/下载和双向同步；远程目录删除也不递归。
